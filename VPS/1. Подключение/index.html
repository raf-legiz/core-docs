    🐧 Подключение к серверу по SSH (Ubuntu)¶

    🔐 Для подключения к серверу используйте следующую команду:

    ssh root@IP -p PORT

    📌 Пояснение параметров:¶

    • root — пользователь с полными правами (суперпользователь)
    • IP — IP-адрес сервера (например: 192.168.1.1)
    • PORT — порт подключения (например: 22)
    ✨ Пример подключения 
    ssh root@192.168.1.1 -p 22

    🧾 При первом подключении появится сообщение:

    Are you sure you want to continue connecting (yes/no/[fingerprint])

    ✔️ Введите:

    yes

    🔑 После этого система запросит пароль:

    root@IP's password:

    ⚠️ Важно: при вводе пароль не отображается — это нормальное поведение системы безопасности SSH.

    2. Смена пароля root, для Ubuntu 24.04¶

    passwd
    Введите новый пароль для root-пользователя (для безопасности, при вводе он не отображается). Затем введите его повторно.

    Комментарии

    • Вводите сложный пароль длинной 15-20 символов. Он должен состоять из заглавных и строчных букв, цифр и спец символов. А чтобы его легко было запомнить составляйте его из парольных фраз не относящихся к вашей жизни.

    • Например, если вы никогда не играли в Minecraft, то подойдёт пароль: Minecraft_eto_moja_jizn’2024!!!2402. Такое легко запомнить и сложно подобрать.

    Смена пароля root, для Ubuntu 26.04

    Если произвели установку Ubuntu 26.04 - при первом входе, он сам попросит сменить пароль. Устанавливаем пароль максимально сложный. Псоле того как установили пароль, система автоматически выйдет и придется войти снова с новым паролем

    3. Создаём нового пользователя¶

    Т.к. доступ у root по ssh к серверу мы закроем, то надо создать нового пользователя, через которого и будем дальше подключаться. Создаём пользователя следующей командой (вместо "user_name" впишите любое имя пользователя, которое хотите и не забудете):

    adduser user_name
    Далее следуем инструкциям на экране, обязательно указывая пароль пользователя (как обычно, при вводе он не отображается). Прочие поля можно оставлять пустыми.

    Предоставляем права sudo пользователю

    usermod -aG sudo user

    где, user - указываем имя созданного пользователя и выполняем команду, если хотим дать доступ спер пользователя

    • Перейти от обычного пользователя к root можно командой: 
      su - root
    • От суперпользователя к обычному: 
      su - user_name

    4. Обновление пакетов¶

    Наличие последней версии системы по умолчанию увеличивает её безопасность (в новых версиях частенько устраняют известные уязвимости). Следующие три команды обновят информацию об актуальных версиях пакетов, обновят их (это займёт некоторое время), а затем перезагрузят сервер (придётся подключиться повторно). Ниже я привел 2 вида:

    Команда full-upgrade (ранее dist-upgrade) выполняет более интеллектуальное обновление: при необходимости она может удалить конфликтующие пакеты или установить новые зависимости.

      apt update && apt dist-upgrade -y && reboot

      apt update && apt full-upgrade -y && reboot
