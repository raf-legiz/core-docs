    # Защищаем соединение по SSH

    Почему важно настроить SSH

    Залётные хакеры первым делом пытаются подключаться к SSH (по стандартному порту 22, от имени root).
    Также Роскомнадзор может догадаться проверять наличие открытого стандартного 22‑го SSH‑порта у вашего сервера.

    Поэтому делаем следующее:

    • Запрещаем подключение по SSH пользователю root (будем входить через нового пользователя)
    • Меняем стандартный порт SSH (с 22 на любой другой)
    • Добавляем подключение по SSH‑ключам и запрещаем вход по паролю

    info "Заходим в настройки ssh-соединения с помощью редактора nano (всё делаем от суперпользователя)"¶

      sudo nano /etc/ssh/sshd_config

    "Убираем #, и изменяем следующие параметры"

    # Port 22
# PermitRootLogin yes
# PasswordAuthentication yes
X11Forwarding yes
# LogLevel INFO
    • УБираем коментарии '#', port можем указать любой. Не используем служебные!!! Используем от 10001 до 65000
    • Отключим вход c root правами по отключив тем самым возможность подключения по ssh через root.
    • Отключим возможность входа по Login пользователя и с root првами.
    Port 45325 # (1)!
PubkeyAuthentication yes # (2)!
PermitRootLogin no # (3)!
PasswordAuthentication no # (4)!
LogLevel VERBOSE # (5)!
X11Forwarding no # (6)!
AllowUsers USER # (7)!
    1. Тут должны придумать порт, указываем тот порт через который будем взаимодействовать с сервером по SSH
    2. Опция дополнительной безопастности, подключаться можем только по PublickKey
    3. Запрещаем подключаться по ssh через root!
    4. Запрещаем подключаться, по логину и паролю. (Если мы тут запрещаем, иногда мы можем подключиться к серверу по логину и паролю. Для того чтобы это исправить читай ниже)
    5. логи
    6. Это способ удалённо запускать графические приложения и видеть их локально через SSH. Данную функцию отключаем.
    7. Разрешаем доступ только определенным пользователям, а остальным запрещаем. Можно прописать несколько пользователей, пишем через пробел.

    Сохраняем файл: 

      Ctrl + O

    Закрываем файл: 

    Ctrl + X

    ВАЖНО !!!

    Если при открытии основного файла конфигурации SSH командой: 'sudo nano /etc/ssh/sshd_config' мы видим строку: 'Include /etc/ssh/sshd_config.d/*.conf'

    это означает, что сервер SSH автоматически подгружает дополнительные конфигурационные файлы из каталога /etc/ssh/sshd_config.d/. Поэтому, чтобы корректно отключить вход по паролю, необходимо перейти в соответствующий файл внутри этой директории и внести изменения именно там. В выбранном конфигурационном файле нужно отключить возможность аутентификации по паролю, установив параметры, запрещающие такой тип входа.

      sudo nano Include /etc/ssh/sshd_config.d/*.conf

    После открытия данного файла, редактируем ее следующим образом:

      PasswordAuthentication yes


      PasswordAuthentication no  # (1)!
    1. мы запрещаем вход по паролю.

    • подгружаем измененные параметры: 
      sudo systemctl reload ssh
    • systemctl restart ssh 
      systemctl restart ssh
